| [content_pipeline.md](content_pipeline.md) | A second registry (sibling of type-wiring): file-name / extension / directory-keyed **content stages** that transform a file's raw bytes around TSV parsing. **Bidirectional** — a *source* direction on read (`decode` decompress → `transcode` XML/JSON/SQLite/`.mtx`→TSV → `macro` COG-expand) and a *sink* direction on export (inverse), with COG present in both (expand on read, **strip COG comments on export**). COG is **format-agnostic**, enabling **data-driven generation of non-data text files** (e.g. Markdown docs from TSV data) at export time. Tracks a per-file **content kind** (`text`/`binary`) so it also handles **binary/asset files**: the load decision is **"does a stage need the bytes?"** not known-vs-unknown, so any binary nothing processes is **never loaded into memory** (`raw_files` passthrough + exporter `copyFileStreamed`), while a binary a matched `asset`-phase stage claims is loaded to re-encode/resize. Collapses the three hard-coded `processContentBV` call sites into one; `exporter.lua` is the sink driver. **Phases 1–5 (plus Phase 4 Part B, reversible gzip) shipped; only the optional Phase 6 remains, deferred until a concrete need appears.** |

| [tables_as_keys.md](tables_as_keys.md) | Why a **table used as a map key** is unsupported, with the code that proves each reason and options to fix. (1) `ltcn`'s grammar makes a table a legal *value* but never a legal *key* (`Key = Number+String+Boolean`), so `[{1,2}]=x` can't re-parse — and no fork adds it (upstream lists it under "Features Not Implemented"); (2) natural JSON keys are always strings (table keys get stringified, unreconstructable) and the type parser rejects a table-typed map key outright; (3) Lua compares table keys by identity, not structure, so content-addressed lookup is meaningless without an interning layer. Key trap: the **serializer writes** `[{1,2}]=v` that nothing can **read back** — asymmetric silent round-trip break. Options: A — make serializers refuse a table key (loud, symmetric; recommended); B — canonical-string key encoding (lossy, doesn't fix natural JSON); C — full value-semantics/interning layer (correct, expensive). |
