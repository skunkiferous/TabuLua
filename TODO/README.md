| [tables_as_keys.md](tables_as_keys.md) | **Option A landed (2026-07-14).** Why a **table used as a map key** is unsupported, with the code that proves each reason, and the fix. (1) `ltcn`'s grammar makes a table a legal *value* but never a legal *key* (`Key = Number+String+Boolean`), so `[{1,2}]=x` can't re-parse — and no fork adds it (upstream lists it under "Features Not Implemented"); (2) natural JSON keys are always strings (table keys get stringified, unreconstructable) and the type parser rejects a table-typed map key outright; (3) Lua compares table keys by identity, not structure, so content-addressed lookup is meaningless without an interning layer. The trap it documented was live: the **serializer wrote** `[{1,2}]=v` that nothing could **read back** — an asymmetric, silent round-trip break. Option A closes it: all four serializers now **refuse** a table key (SQL export inherits it), reported through `badVal` as an ordinary bad value — naming file, row and column — because such a value can only come from an `=expr`, a pre-processor or a transcoder, and a raised error would abort the load anonymously. Two things the plan missed: `unquotedStr` wrappers are tables in the key position (that is how `[3]=v` / `[true]=v` are written at all) and had to stay legal; and `badVal` rendered its bad value with the very serializer being made strict, *unprotected* — a pre-existing latent crash for recursive/too-deep tables, now degraded to `<unserializable table: reason>`. B (canonical-string keys, lossy, doesn't fix natural JSON) and C (value-semantics/interning layer, correct, expensive) stay parked. |

| [boxed_int64.md](boxed_int64.md) | **All phases landed (2026-07-21).** `int64` is now an interned, immutable **box** — an empty table carrying a `"int64"` metatable, payload held in a module-private weak map (a native 64-bit int on Lua 5.3+, an FFI `int64_t` cdata on LuaJIT) — replacing the former canonical-decimal-string. The point: exports are **exact on every Lua version including LuaJIT** (a double-only runtime), and a serializer can recognize an int64 **by value** (`int64.is`, the only correct test — `type` says `"table"`, `math.type` says `nil`) with no schema, so it survives inside **untyped containers, arrays and map keys**.
